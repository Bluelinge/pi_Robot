# 新树莓派快速部署清单

适用场景：

- 新的树莓派板子
- 系统为 `Raspberry Pi OS (64-bit)`
- 先验证 `D435i`，再部署网页查看和机器人主控
- 当前项目目录为 `pi_robot`

这份文档按“先跑通、再完整部署”的顺序写。主线优先使用：

- `uv + Python 3.12`
- `pyrealsense2` 直接验证 `D435i`
- 先启动 `camera_viewer`
- 再进入 `pi_robot.main`

## 0. 先准备硬件

- 树莓派电源尽量稳定，建议官方或稳定 `5V 3A` 级别
- `D435i` 接到树莓派蓝色 `USB 3.0` 口
- 第一次部署时，先不要接 `ESP32` 也可以
- 树莓派和 Windows 电脑放在同一局域网

## 1. 系统初始化

系统写入完成并首次开机后，在树莓派终端执行：

```bash
sudo apt update
sudo apt full-upgrade -y
sudo systemctl enable --now ssh
hostname -I
```

记下树莓派 IP，后面 Windows 用它来复制代码和打开网页。

如果你想确认 `D435i` 已被系统识别：

```bash
lsusb | grep -i intel
```

## 2. 安装 Python 3.12 环境

不要把 `Python 3.13` 当主线。`pyrealsense2` 在 `aarch64` 上与 `Python 3.12` 更稳。

先安装 `uv`：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

安装 `Python 3.12` 并创建虚拟环境：

```bash
~/.local/bin/uv python install 3.12
~/.local/bin/uv venv --python 3.12 --seed ~/rsenv312
source ~/rsenv312/bin/activate
python --version
python -m pip --version
```

验收标准：

- `python --version` 显示 `3.12.x`
- `python -m pip --version` 正常输出

## 3. 先单独验证 D435i

在虚拟环境里安装 `pyrealsense2`：

```bash
source ~/rsenv312/bin/activate
python -m pip install --upgrade pip
python -m pip install pyrealsense2==2.57.7.10387
```

运行最小验证脚本：

```bash
python - <<'PY'
import pyrealsense2 as rs
print("module:", rs.__file__)

pipeline = rs.pipeline()
config = rs.config()
config.enable_stream(rs.stream.color, 424, 240, rs.format.bgr8, 15)
config.enable_stream(rs.stream.depth, 424, 240, rs.format.z16, 15)

profile = pipeline.start(config)
frames = pipeline.wait_for_frames()
print("color ok:", bool(frames.get_color_frame()))
print("depth ok:", bool(frames.get_depth_frame()))
print("device:", profile.get_device().get_info(rs.camera_info.name))
pipeline.stop()
PY
```

验收标准：

- `color ok: True`
- `depth ok: True`
- `device: Intel RealSense D435I`

如果这里不通过，不要继续部署 `pi_robot`，先把 `D435i` 本身连通问题解决。

## 6. 先启动相机网页服务

这是新板首选入口，因为它不依赖 `ESP32` 串口。

## 7. 再进入主控服务

只有当 `D435i` 网页画面正常后，才建议启动主控服务。

先生成配置文件：

```bash
source ~/rsenv312/bin/activate
cd ~/pi_robot
cp .env.example .env
```

当前配置模板在 [.env.example](E:/AI/Codex/RoBot/Project_Develop/pi_robot/.env.example)。

编辑：

```bash
nano .env
```

至少确认这些值：

```env
PI_ROBOT_HOST=0.0.0.0
PI_ROBOT_PORT=8000
PI_ROBOT_CAMERA_VIEWER_PORT=8001
PI_ROBOT_SERIAL_PORT=/dev/ttyUSB0
PI_ROBOT_SERIAL_BAUD=115200
PI_ROBOT_COLOR_WIDTH=424
PI_ROBOT_COLOR_HEIGHT=240
PI_ROBOT_DEPTH_WIDTH=424
PI_ROBOT_DEPTH_HEIGHT=240
PI_ROBOT_CAMERA_FPS=15
PI_ROBOT_REALSENSE_REQUIRE_IMU=false
```

如果 `ESP32` 还没接，先不要启动 `pi_robot.main`。  
如果已经接上，先确认串口名：

```bash
ls /dev/ttyUSB* /dev/ttyACM* 2>/dev/null
```

把 `.env` 里的 `PI_ROBOT_SERIAL_PORT` 改成实际设备名，例如：

- `/dev/ttyUSB0`
- `/dev/ttyUSB1`
- `/dev/ttyACM0`

然后启动主控服务：

```bash
source ~/rsenv312/bin/activate
cd ~/pi_robot
python -m pi_robot.main
```

浏览器打开：

```text
http://<树莓派IP>:8000
```

## 8. 配置开机自启 systemd

只有当手动启动已经成功后，才做这一步。

当前仓库里自带的 [deploy/robot.service](E:/AI/Codex/RoBot/Project_Develop/pi_robot/deploy/robot.service) 还是旧示例：

- `WorkingDirectory=/opt/pi_robot`
- `ExecStart=/usr/bin/python3 -m pi_robot.main`
- `User=pi`

新板不要直接照抄，应该按实际环境改成：

```ini
[Unit]
Description=Pi Robot Control Service
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/home/admin/pi_robot
EnvironmentFile=/home/admin/pi_robot/.env
ExecStart=/home/admin/rsenv312/bin/python -m pi_robot.main
Restart=always
RestartSec=2
User=admin

[Install]
WantedBy=multi-user.target
```

保存到项目目录，例如：

```bash
cat > /home/admin/pi_robot/robot.service <<'EOF'
[Unit]
Description=Pi Robot Control Service
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/home/admin/pi_robot
EnvironmentFile=/home/admin/pi_robot/.env
ExecStart=/home/admin/rsenv312/bin/python -m pi_robot.main
Restart=always
RestartSec=2
User=admin

[Install]
WantedBy=multi-user.target
EOF
```

安装并启用：

```bash
sudo cp /home/admin/pi_robot/robot.service /etc/systemd/system/robot.service
sudo systemctl daemon-reload
sudo systemctl enable --now robot.service
sudo systemctl status robot.service
```

如果你以后只想让相机网页自启，也可以把 `ExecStart` 改成：

```bash
/home/admin/rsenv312/bin/python -m pi_robot.camera_viewer
```

## 9. 退出方法

前台运行时退出：

```text
Ctrl + C
```

查询端口是否还在监听：

```bash
ss -ltnp | grep 8001
ss -ltnp | grep 8000
```

后台停止相机网页：

```bash
pkill -f "pi_robot.camera_viewer"
```

后台停止主控服务：

```bash
pkill -f "pi_robot.main"
```

如果是 `systemd` 启动的，停止方式是：

```bash
sudo systemctl stop robot.service
```

## 10. 常见故障快速排查

### 1. `pyrealsense2` 找不到版本

表现：

```text
No matching distribution found for pyrealsense2
```

处理：

- 不要用系统 `Python 3.13`
- 切到 `uv + Python 3.12`

### 2. `camera_viewer` 打不开

先不要急着排网页，先单独验证相机：

```bash
source ~/rsenv312/bin/activate
python - <<'PY'
import pyrealsense2 as rs
pipeline = rs.pipeline()
config = rs.config()
config.enable_stream(rs.stream.color, 424, 240, rs.format.bgr8, 15)
config.enable_stream(rs.stream.depth, 424, 240, rs.format.z16, 15)
pipeline.start(config)
frames = pipeline.wait_for_frames()
print("color ok:", bool(frames.get_color_frame()))
print("depth ok:", bool(frames.get_depth_frame()))
pipeline.stop()
PY
```

### 3. `pi_robot.main` 报串口不存在

先确认真实串口名：

```bash
ls /dev/ttyUSB* /dev/ttyACM* 2>/dev/null
```

再修改 `.env` 的 `PI_ROBOT_SERIAL_PORT`。

### 4. 网页端口不通

先看服务是否在运行：

```bash
ss -ltnp | grep 8001
ss -ltnp | grep 8000
```

再确认树莓派 IP：

```bash
hostname -I
```

## 11. 不要走的坑

- 不要默认用系统 `Python 3.13` 安装 `pyrealsense2`
- 不要把 `librealsense` 源码编译当主线
- 不要先启动 `pi_robot.main` 再验证相机
- 不要在未确认串口名时，把 `.env` 固定写成 `/dev/ttyUSB0`

## 12. 推荐部署顺序总结

推荐顺序固定为：

1. 写入系统
2. 联网、开 SSH
3. `uv + Python 3.12`
4. `pyrealsense2` 验证 `D435i`
5. 复制 `pi_robot`
6. 跑 `camera_viewer`
7. 再配 `ESP32` 串口
8. 跑 `pi_robot.main`
9. 最后再上 `systemd`
